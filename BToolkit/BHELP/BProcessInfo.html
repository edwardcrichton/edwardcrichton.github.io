BProcessInfo

System/Window crashes etc. may result in B processes left `hanging', which may, in turn, affect the ability to run the B-Toolkit because of the licensing mechanism. Running the script `BProcessInfo' will display the user's B processes currently running (as well as those of other users) together with an invitation to kill the user's processes; where possible, a request to the BToolkit daemon BToolkitd will be made to remove any licences freed as a result.

The script resides in the $BKIT directory and is, of course, invoked outside the B-Toolkit.
